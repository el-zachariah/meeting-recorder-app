# Meeting Recorder v0.6.0 release evidence/signoff

Status: QA-accepted implementation candidate; publish only after commit, CI, final Zachariah verification, and downloaded-asset checks.

## Scope

- Persistent GUI settings in XDG config for default save location, transcriber model, capture defaults, Obsidian defaults, and post-recording toggles.
- Dropdown UI with visible Record, Pause, Resume, and Stop controls plus Settings access.
- Timestamp-based default meeting name/folder title.
- Settings panel for default save location and transcriber settings.
- Best-effort pause/resume support using Linux process pause/resume for ffmpeg, with pause metadata written to saved meeting metadata.
- COSMIC-aware tray/status diagnostics without claiming native COSMIC integration; GUI remains generic AppIndicator/Tk.

## Known limitations to carry into release notes

- Normal GUI startup requires a desktop tray/AppIndicator stack and a desktop environment that displays tray icons.
- On Pop!_OS COSMIC, Meeting Recorder currently uses generic AppIndicator/tray support and Tk; it is COSMIC-aware in diagnostics but not native COSMIC UI.
- Pause/resume is a best-effort process pause (`SIGSTOP`/`SIGCONT`) rather than segment-level recording composition.
- Screen video capture remains X11/XWayland-oriented; audio-first recording remains the default.
