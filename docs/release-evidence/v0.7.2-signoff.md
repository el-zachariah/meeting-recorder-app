# Meeting Recorder v0.7.2 release signoff

Date: 2026-06-05
Commit: ba98b52

## Product shape

- Runtime contract: `meeting-recorder gui` is a native system-tray application.
- The root/main Tk window is hidden.
- The tray/default action opens the recorder dropdown.
- The dropdown remains the operational surface for record, pause/resume, stop/save, configurable default save path, transcription, and summary workflow.
- The modern HTML/CSS renderer remains release/design evidence only until the modern visual treatment is rebuilt inside the tray/dropdown contract.

## Engineer signoff

The engineering gate restores the last sensible product fundamentals from the v0.6.0 line: native system tray, hidden root window, tray-opened dropdown, persisted settings/default path, and existing post-recording transcription/summarization workflow. Regression tests now fail if `meeting-recorder gui` bypasses `SystemTrayDropdownGUI` or calls the standalone WebView launcher.

## QA/research signoff

The QA gate confirms the v0.7.0/v0.7.1 standalone Electron/Chromium window direction was a product-shape regression. v0.7.2 intentionally restores the system-tray/dropdown runtime and treats the modern WebView screenshot renderer as evidence/design work only, not the app shell.
